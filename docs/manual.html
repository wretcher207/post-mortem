The Panel Interface

The Post Mortem panel is a dockable ReaImGui window inside REAPER. It is a thin client: it writes JSON job files and renders the results that come back from the local sidecar engine. No analysis runs in the panel itself.

The mockups below are grounded in the actual panel source (post_mortem_panel.lua) and use the real Dead Pixel palette: #0a0a0a background, #e8e6e1 text, #a83232 primary buttons, #c99a3f amber for uncertainty, #3f9c8a teal for passed checks.

01

What Post Mortem Reads

Post Mortem reads the selected track's full state at the moment you run a Track Check. Nothing is cached from earlier sessions. If you change a parameter and check again, it reads the new value.

FX Chain
Every plug-in, every parameter
Name, enabled/bypassed state, and the full parameter list with current normalized values and formatted display values (like 120 Hz or -4.50 dB). Parameter count is read per plug-in, with a parameters_truncated flag if the API cut off.
Routing
Routing matrix, not guesswork
Sends, receives, parent bus, fader level, pan, phase, and automation mode. The volume field is converted from linear to dB in the bridge before the model sees it.
Capture
Post-FX stem render
A short section of the track's post-fader, pre-master output rendered to a temporary WAV. Default is 10 seconds from the edit cursor. The file is measured and then removed. Raw audio never leaves your computer.
Project
Session context
Project name, tempo, and time signature. Gives the model genre context so it doesn't suggest a jazz move on a death metal session.
The capture path

Post Mortem uses REAPER's native stems render (not solo) to capture the track's post-FX, post-fader output, pre-master. This means no solo state to corrupt, and no master bus coloration printed into the capture. A stems render of a child track does not include parent bus FX. The model sees the parent name so it knows a parent chain exists; analyzing it is not in 0.1.0.

02

Audio Measurements

Post Mortem computes the spectrum and dynamics from the captured WAV in pure numpy. No librosa, no real-time analyzer. The measurements are what the model sees instead of raw audio.

Measurement What it tells the model Unit
integrated_lufs Overall loudness of the capture LUFS
sample_peak_db Highest sample value, headroom reference dBFS
true_peak_db Inter-sample peak from REAPER RENDER_STATS dBFS
rms_db Average level, perceived loudness reference dBFS
crest_factor_db Peak minus RMS, dynamic range indicator dB
loudness_range_lu Loudness variation across the capture LU
lufs_momentary_max Peak 400ms loudness moment LUFS
lufs_short_term_max Peak 3-second loudness window LUFS
silence_fraction Share of 100ms blocks below -70 dBFS 0.0 to 1.0

1/3-Octave Spectrum

31 bands from 20Hz to 20kHz, computed as a combined-channel power-domain average (not an amplitude mono-sum). This avoids the phase cancellation trap where an out-of-phase stereo stem would amplitude-sum to silence and report a false diagnosis. L/R asymmetry is not visible in this single band array. That is a v2 field, not a schema change.

20Hz 100Hz 500Hz 1kHz 5kHz 10kHz 20kHz

Stereo Measurements

correlation
Zero-lag L/R phase correlation. null for mono tracks. A positive value means the channels are in phase. A negative value means something is flipped.
mid_rms_db / side_rms_db
Mid (L+R) and side (L-R) energy. A wide side relative to mid means a lot of stereo content. A near-zero side means it's basically mono.
balance_db
L-minus-R balance. Positive means left-heavy, negative means right-heavy. Useful for catching a panned track that's feeding the wrong bus.

Capture isolation gate

Verified isolation required

Post Mortem only diagnoses a track when the bridge proves the capture is that track alone. capture_scope: "isolated_track" with isolation_verified: true is mandatory evidence. A full-mix render, unverified capture, or missing provenance is refused before the model is called. Soloing a track by hand does not change this gate.

Item-less routing and bus tracks are supported by the current isolation path. Ordinary tracks containing media items may be refused rather than diagnosed from a full-mix render. That refusal is a product feature, not a bug.

Silence gate

If the capture's overall RMS is at or below -60 dBFS, or if 85% or more of the capture is below -70 dBFS, Post Mortem refuses to spend a model call on dead air. It tells you to park the cursor where the track is playing and exits. --force overrides the silence gate. It never overrides capture isolation.

04

Understanding the Diagnosis

The diagnosis has two parts: a finding (what's wrong) and a proposal (one concrete move). Both are validated against the measured data before they reach you.

Finding

Field Description
summary Two plain-language sentences. What the numbers say. text
probable_cause Which FX, parameter, or routing is likely causing it. text
confidence How much signal vs guessing. Low, medium, or high. enum
evidence_refs Exact payload paths the finding cites. Verified to exist. list

Confidence levels

High
Strong measured signal. The evidence directly supports the finding.
Medium
Some signal, some inference. The finding is a reasoned guess, not a bluff.
Low
Weak evidence. The model is saying "I'm not sure" with a reason.
When confidence is capped

A silence fraction of 0.75 or higher caps confidence at low and rejects any actionable proposal. A response that introduces a cross-track claim in the single-track path is replaced with a safe low-confidence finding and rejection_reason: "cross_track_claim".

Proposal operations

The proposal is one move, not five. If the model proposes something the validator can't support, the operation becomes none with a rejection reason, and the finding still reaches you as advice.

01
set_track_volume
unit: db
Fader level change. Current and proposed values in dB.
02
set_track_pan
unit: normalized_pan (-1.0 to 1.0)
Pan position change. Left is negative, right is positive.
03
set_fx_param
unit: normalized (0.0 to 1.0)
One numeric FX parameter, identified by verified index and name.
04
set_fx_bypass
unit: boolean
Toggle bypass on one plug-in, identified by GUID.
What proposals never do in 0.1.0

Add or remove plug-ins, rewrite routing, edit items, or write automation. The first release supports four operations. Everything else stays as advice.

Guardrails

Preview computes before/after deltas and runs five safety checks. These do not call the preview "better." They flag risks so you can decide.

new_clipping
Fails above -0.3 dBFS when the baseline was below. Pre-existing clipping is not "new."
loudness_shift
Warns above 3 LU change. The A/B becomes unfair so the intended-direction sentence is blocked.
phase
Warns on a 0.25 correlation drop or a positive-to-negative flip.
stereo_balance
Warns above 3 dB L/R balance shift.
silence
Fails at 0.75 or higher silence fraction. Nothing else is claimed.

Outcome sentences

Exactly one of these three locked sentences is shown. No code path can claim improvement without the goal metric moving past its noise floor in the intended direction. The word "better" never appears.

✓
"The candidate moved in the intended direction."
~
"The measurement changed, but not enough to prove improvement."
✕
"This introduced a new risk; keeping the original is safer."

Noise floors

Each metric has a below-this-it-counts-as-no-change floor. If the delta doesn't clear the floor, the outcome is "not enough to prove improvement," not "it worked."

Metric Noise floor
Level / loudness0.5 dB / 0.5 LU
Correlation0.05
Spectrum band1.0 dB per band

A/B controls

Original
Play the baseline capture. This is the track's current state, unchanged.
Suggested
Play the candidate capture. The proposed value is already applied and restored. You're hearing the rendered result, not a live change.

A loudness-match toggle is available for subjective comparison. The exact change, goal metric, and guardrail results are shown alongside the A/B.

Adjust

If the proposed value is close but you want to nudge it, the Adjust slider offers a conservative range around the proposed value, clamped by the same move-limit constants the validator uses on proposals. You cannot type a value outside the safe range through the panel or the sidecar.

06

Apply Fix

Apply performs a fresh identity check (track GUID, FX GUID, parameter index, parameter name, current value) and creates one named REAPER undo point. One Ctrl+Z (or Cmd+Z) returns the project to the previous state.

What Apply guarantees

The value is applied only if every identity field still matches the diagnosis. If anything changed between preview and apply, Apply refuses and tells you to run a fresh Track Check.

One undo point
Identity rechecked
No automation written
No routing changes
No items edited
No plug-ins added or removed

Onboarding

Step 1: Welcome

Three trust statements before anything happens:

No project changes during analysis
Track Check is read-only. Nothing in your project moves.
Preview changes are temporary
Fix Preview restores the original before returning control to you.
Raw audio stays local
The model sees measurements. Not your stems, not your WAV files.

Step 2: Automatic setup

Found REAPER
Setup detects the REAPER resource directory. Portable installs are supported via manual folder selection.
Installed the local bridge
Reaper Daemon is installed into the REAPER resource folder. No admin access needed.
Enabled safe track capture
Requires one REAPER restart. This changes bridge capture permission, not the project.
Added the Post Mortem panel
Panel action is registered in REAPER's Actions list. Search "Post Mortem" to open it.

If the render dialog requires "Automatically close when finished," Setup shows one illustrated instruction and a Test Again button. This is the only manual setup exception.

Step 3: Connect analysis

Option A
Connect an API key
Paste an Anthropic API key. One small validation request is made before the key is saved locally. The key lives in your Post Mortem config, not in the project.
Option B
Use an MCP client
The model in your MCP client performs the diagnosis. Post Mortem does not need a separate API key. The client needs a working Reaper Daemon MCP connection.
Advanced
Compatible endpoint
Anthropic-compatible providers use POSTMORTEM_API_KEY, ANTHROPIC_BASE_URL, and POSTMORTEM_MODEL. The provider key is kept separate from any Anthropic key so the wrong credential is never forwarded to another host.

Step 4: First success

Select a track, put the edit cursor over audio, and run the first 10-second Track Check. Onboarding is complete only when a real project track has produced a diagnosis.

09

Command Line Interface

The free MIT engine ships with a CLI for developers and terminal users. Same engine, same hedge contract, same isolation gate. No panel, no installer, no license check.

Basic usage

# Single-track diagnosis (text output)
postmortem "Kick"

# JSON output for preview/commit pipeline
postmortem "Kick" --format json

# Cross-track masking (2+ tracks, prose output)
postmortem "Kick" "Bass"

Options

Flag Description Default
--seconds N Capture length in seconds 10
--keep-wav Keep the local temporary stem instead of deleting it off
--payload-only Print measured data, skip the model call. Never gated by silence. off
--format text|json Output format. JSON is single-track only. text
--force Bypass the silence gate only. Never bypasses capture isolation. off

Fix Preview from the terminal

# Save a diagnosis to file
postmortem "Kick" --format json > diagnosis.json

# Preview the proposed fix (captures baseline + candidate)
postmortem preview diagnosis.json

# Apply the fix (creates one undo point)
postmortem commit diagnosis.json

Local data paths

Platform Data root
macOS~/Library/Application Support/PostMortem
Windows%APPDATA%\PostMortem
Linux$XDG_DATA_HOME/postmortem

Provider config (API keys) is separate: ~/.config/postmortem/config on macOS and Linux, stored with owner-only permissions where the OS supports them. Environment variables can be used instead.

License checks

The paid panel uses a signed JSON license. Validation uses the public key in the installed product and does not contact a license server. The local license record contains the holder, product, purchased major version, issue date, update-entitlement date, signing-key ID, and signature. The purchased major version does not expire. The update-entitlement date is a separate check with a 30-day offline grace period.

Telemetry

None

Version 0.1.0 has no analytics, no usage telemetry, no remote crash reporting, no account system, and no automatic license check. The only normal network request is the model-provider call initiated for analysis, or traffic initiated by your separate MCP client.

Feedback

The Not Helpful action appends a local record to feedback.jsonl. Version 0.1.0 does not upload that file.

Troubleshooting

Start with the exact message the panel or Setup shows. Recovery messages are part of the product. Follow them before changing files by hand.

Panel not in Actions list
Quit REAPER. Run Setup, choose Install or Update. Restart REAPER. Search "Post Mortem" in Actions.
Engine not running
Choose Start Engine in the panel. If it won't stay running, open Setup and choose Test Setup. Reinstall if files are incomplete.
Needs REAPER
REAPER must be open with Reaper Daemon running. Restart REAPER once after install or update. If it persists, run Test Setup.
Safe capture not enabled
Choose Enable Safe Capture in the panel, then restart REAPER. This changes bridge capture permission, not the project.
Render window stays open
In REAPER's render window, enable "Automatically close when finished." REAPER remembers it. Return to panel and choose Test Again.
Capture is silent
Move the edit cursor to where the track is making sound. Muted items, empty time selection, or a quiet routing track can all produce silence.
Capture isolation refused
The bridge can prove isolation for item-less routing and bus tracks. A track with media items may be refused. Soloing does not bypass this.
API key rejected
Confirm the key is active with its provider. Check billing or credits. Third-party endpoints need POSTMORTEM_API_KEY, not ANTHROPIC_API_KEY.
License not accepted
Use the original signed JSON file, unedited. Confirm it is for version 1. Check the Mac's date and time. Reopen Setup, Add License.
Preview won't play
Preview playback uses SWS. Reinstall with Setup if the panel says SWS is missing. Existing user-installed SWS or ReaImGui is not overwritten.
Preview didn't confirm restoration
Do not Apply. Check the affected parameter in REAPER. The bridge owns crash recovery and normally restores. If still changed, use undo and save under a new name.
12

Known Limits

Platform
Apple silicon macOS only
Windows and Linux paid installers are withheld until clean-machine customer checks pass. The free engine and Reaper Daemon support all three platforms.
Capture
Item tracks may be refused
The bridge can prove isolation for item-less routing and bus tracks. Ordinary tracks with media items may fail the gate. This is a product feature.
Scope
Track Check and Fix Preview only
Mix Check, history, hosted credits, and automatic update checks are not in 0.1.0.
Operations
Four supported moves
Volume, pan, FX bypass, one numeric FX parameter. No plug-in add/remove, routing edits, item edits, or automation.
Spectrum
1/3-octave resolution
31 bands. A narrow resonance at 2.4kHz shows as a bump in the 2.5kHz band. Higher-resolution FFT is a v2 addition.
Stereo
Single band array
L/R asymmetry is not visible in the combined spectrum. Per-channel spectra are a v2 field addition, not a schema change.
Parent bus
Parent FX not analyzed
A stems render of a child track does not include parent bus FX. The model sees the parent name. Analyzing it is not in 0.1.0.
Real-time
Static capture only
Post Mortem renders a short section and measures it. There is no real-time spectrum, no live monitoring, no audio feedback loop to the model.
